# Dialogue Generate

Step-by-step process for creating a two-column dialogue exercise. Follow these steps in order — do not skip to the table format.

## 1. Define the scene

Before writing any lines, establish:
- Who are the two speakers? (names, basic traits)
- Where are they? (context for the meeting)
- What do they know about each other? (strangers, colleagues, etc.)
- What vocabulary/grammar is this dialogue meant to practice?

## 2. Write the conversation in sequence

Write the full dialogue as a numbered list — Speaker A, then Speaker B, alternating. NOT in table format yet.

Rules while writing:
- **Names at the greeting.** "Oi, sou o/a [name]" — nobody volunteers their name after "how are you?"
- **Reciprocity.** Both speakers share information. If A asks "where are you from?", B answers AND asks back. A dialogue where one person only asks is an interview, not a conversation.
- **No implied knowledge.** A stranger cannot react to specifics they haven't been told. "O novo professor da escola?" is impossible if nobody mentioned a school.
- **Natural transitions.** Don't jump from origin to goodbye. Each topic shift needs a bridge (e.g., "E você?" or "Legal!").
- **Only taught vocabulary.** Every word must appear in the vocab box or have been taught earlier in the worksheet.

## 3. Verify the conversation

Read the sequential version out loud (or mentally). For each line, ask:
- Does this answer what the previous speaker just said?
- Would a real person say this in this situation?
- Does the speaker know enough to say this? (no telepathy)

If any line fails, fix it now — before touching the table.

## 4. Convert to two-column table

The table format: left column speaks FIRST, right column RESPONDS.

Each table row contains TWO turns — left's line and right's reply. So a 12-line conversation becomes 6 table rows.

Map the sequential lines to rows:
- Row 1: lines 1 (left) and 2 (right)
- Row 2: lines 3 (left) and 4 (right)
- Row 3: lines 5 (left) and 6 (right)
- etc.

The student fills in the left column. The right column is printed.

## 5. Build the answer key

One answer per table row, numbered to match. The answer is whatever goes in the left column (the blank).

Count: number of answer key items MUST equal number of table rows.

## 6. Build the vocab box

List every word used in the dialogue that hasn't been defined earlier in the worksheet. No word should appear in exercises without a definition.

## Lessons learned

- A dialogue survived 15+ rounds of edits without anyone noticing one speaker never answered a single question about himself — because nobody read it as a conversation.
- A rewritten dialogue had the answer key off by one row because alignment wasn't traced row by row from the sequential version.
- "O novo professor da escola?" implied knowledge that hadn't been established — Luis couldn't know about a specific school position from a stranger.
- "Bem, obrigado. Eu sou o Luis. E você?" — nobody volunteers their name after "how are you?" Names belong at the greeting.
